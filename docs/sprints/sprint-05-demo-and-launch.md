# Sprint 05 - Demo and Launch Hardening

## Prompt

You are working on AgentMarket as a demo-first product. The core product value is visible only if the orchestration run is smooth, the savings story is legible, and fallback modes are explicit.

This sprint is about making the product presentation-ready.

### Goal

Turn the current MVP into a reliable demo build with crisp UX, stable fallback paths, and a clear launch checklist.

### Scope

- Improve the orchestration experience for a live demo.
- Make the savings story impossible to miss.
- Add or refine replay and mock modes if needed.
- Tighten visual polish only where it strengthens comprehension.
- Produce a short operator checklist for running the demo.

### Priority areas

- orchestration loading and completion states
- savings panel clarity
- task event progression
- empty states and error states
- demo goal presets
- replay or forced routing controls
- deployment and operator docs

### Constraints

- Do not turn this into a design rewrite sprint.
- Do not add broad new features unrelated to the demo path.
- Keep changes focused on the critical narrative: route smarter, spend less.

### Acceptance criteria

- A teammate can run the app and understand the demo flow without oral explanation.
- The savings counter and routing logic are visually obvious.
- There is a fallback path when live model behavior is unstable.
- Deployment and demo steps are documented clearly.

### Suggested work plan

1. Walk the current demo flow from homepage to orchestration run.
2. Identify confusion points and weak transitions.
3. Improve only the screens and states that affect the main story.
4. Add a short demo script and launch checklist.
5. Validate the app in the exact mode intended for presentation.

### Definition of done

The product can survive a live demo without relying on luck.
